⚔️ ATTACK MODE (S2 System)

Aggressive profit-making using edge-based entries

  • Wait 5 minutes into 15-min window
  • Check BTC position - above or below strike?
  • Calculate edge from 5-year persistence data
  • Enter if edge ≥ 10%
  • Dynamic bet sizing based on edge strength
Edge Calculation:
edge = historical_persistence - kalshi_price

Example at minute 5:
History says BTC stays in direction 71.3% of time
Kalshi prices it at 55c (55% implied)
Edge = 71.3% - 55% = 16.3% edge
Dynamic Bet Sizing:
scale = (edge - 10%) / 20% # 0 to 1
bet_pct = 2% + scale × 6% # 2% to 8% of bankroll

Examples:
• 10% edge → 2% of bankroll
• 20% edge → 5% of bankroll
• 30%+ edge → 8% of bankroll

🛡️ RECOVER MODE (S3 System)

Conservative recovery using high win-rate entries

  • Wait 10 minutes into 15-min window
  • Only enter at 80-92c (sweet spot)
  • 90%+ win rate at these prices
  • Martingale sizing to recover losses
  • Max 2 recovery attempts then reset
Recovery Bet Calculation:
recovery_target = total_loss × 1.10 # +10% buffer
net_profit_per_contract = (100¢ - entry) - fee
contracts = recovery_target / net_profit_per_contract

Example:
Lost $50 → Need to recover $55 (with buffer)
Entry at 85c → Net profit = 15c - 1c fee = 14c
Contracts needed = $55 / $0.14 = 393 contracts
Why 80-92c?
• At 10 min mark, persistence is 84%+
• Kalshi prices at 80-92c = slight edge
• Combined = 90%+ actual win rate
• Safe for recovery!
